 Photon-WASM 性能优化审计报告

  1. 项目概述

  Photon 是一个高性能图像处理库，同时支持本地和 WebAssembly (WASM)
  环境。代码库已经包含了一些性能优化工作，包括 SIMD 加速和部分算法改进。

  2. 已实现的优化

  2.1✅ SIMD 优化
   - 位置: crate/src/simd.rs
   - 已实现的函数:
     - alter_channel_simd - 通道调整 (使用 128-bit SIMD 向量)
     - alter_channels_simd - 多通道调整
     - inc_brightness_simd / dec_brightness_simd - 亮度调整
     - invert_simd - 颜色反转
     - adjust_brightness_simd - 统一的亮度调整
   - 特点: 使用 std::simd 模块，每次处理 16 个通道值

  2.2 算法优化
   - 卷积模糊: 高斯模糊使用线性时间算法
   - 对比度调整: 使用查找表 (LUT) 优化
   - Gamma 校正: 使用查找表预先计算

  3. 性能优化机会

  3.1✅ 高优先级优化

  3.1.1 扩展 SIMD 覆盖范围
  文件: crate/src/simd.rs

  当前状态: 仅覆盖基础的通道操作

  建议优化:
   1. 灰度转换: monochrome.rs:grayscale - 当前为标量循环，可使用 SIMD
      加速
      // 现有实现: 单像素遍历
      for i in (0..len).step_by(4) {
          let avg = (pixels[i] + pixels[i+1] + pixels[i+2]) / 3;
          // ...
      }

   2. 阈值化: monochrome.rs:threshold - 可 SIMD 化亮度计算

   3. 去饱和度: monochrome.rs:desaturate - RGB 排序可用 SIMD 优化

  预期收益: 2-4x 加速 (对于大图像)

  3.1.2✅ 优化 ImageIterator 使用
  文件: crate/src/iter.rs

  问题: ImageIterator 使用除法和取模计算坐标，性能不佳
   fn next(&mut self) -> Option<Self::Item> {
       let n = self.item;
       self.item += 1;
       if n < (self.width * self.height) {
           Some((n / self.height, n % self.height))  // 昂贵的除法
       } else {
           None
       }
   }

  建议:
   - 使用线性索引直接访问像素
   - 对于必须使用 (x, y) 坐标的地方，使用嵌套循环替代迭代器
   - 或实现行优先迭代器以避免除法

  受影响函数:
   - channels.rs:selective_hue_rotate
   - channels.rs:selective_* 系列函数
   - effects.rs:halftone
   - effects.rs:multiple_offsets

  预期收益: 10-30% 加速

  3.1.3 ✅减少内存分配

  问题点:

   1. 重复的 DynamicImage 转换 (helpers.rs:dyn_image_from_raw)
      - 每次调用都创建新的 ImageBuffer
      - 在 colour_spaces.rs 中频繁使用

     建议: 对于只读操作，直接在像素数组上工作

   2. padding 函数中的 Vec 分配
      - transform.rs:padding_uniform 等函数
      - 使用 push 逐字节添加

     建议: 预分配 Vec::with_capacity() 或使用 copy_from_slice

   3. 查找表重复创建
      - effects.rs:adjust_contrast 每次调用都创建 LUT
      - colour_spaces.rs:gamma_correction 每次调用都创建 gamma 表

     建议: 对频繁调用的函数考虑缓存 LUT

  预期收益: 减少内存分配开销，降低 GC 压力 (WASM)

  3.1.4 ✅优化卷积操作
  文件: crate/src/conv.rs

  当前问题:
   - 使用 image crate 的 filter3x3 通用实现
   - 没有利用 WASM SIMD

  建议:
   1. 为常用卷积核实现 SIMD 版本
   2. 特别优化: box_blur (已部分优化)、sharpen、edge_detection

  预期收益: 1.5-2x 加速

  8. 总结

  Photon-WASM 已经有良好的基础架构，SIMD 模块已实现。主要优化机会在于:

   1. 扩展 SIMD 覆盖到更多算法
   2. 优化迭代器使用以避免昂贵的除法运算
   3. 减少内存分配特别是查找表和临时对象
   4. 修复现有 bug (primary 函数)
